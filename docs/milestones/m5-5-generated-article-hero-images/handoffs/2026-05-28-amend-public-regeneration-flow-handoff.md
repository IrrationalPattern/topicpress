## Amended Contract Summary

- OpenAI remains the only MVP image provider.
- Use one public Supabase Storage bucket: `article-hero-images`.
- Do not require or create the private `article-hero-image-candidates` bucket in the amended MVP.
- Generate or intentionally regenerate one current hero image for an article in review.
- Store generated bytes in `article-hero-images`.
- Update `articles.hero_image_url` on successful generation/regeneration.
- Do not regenerate on page load or during publish.
- Do not block article ready/publish solely because no generated image exists.
- Public routes continue to show only published articles and render the image through `articles.hero_image_url`.

## Follow-Up Implementation

1. Revise the DB/storage contract so successful generated image metadata can carry a public URL immediately.
2. Rewrite the worker image-generation path to use `article-hero-images` directly and update `articles.hero_image_url`.
3. Remove image approval/promotion requirements from article review and publish gates.
4. Replace approve/reject image UI with explicit generate/regenerate controls.
5. Rerun T007 QA against the amended flow.
